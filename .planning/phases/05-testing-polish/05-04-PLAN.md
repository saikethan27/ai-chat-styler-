# Plan 05-04: Extension Icons

## Objective

Create a complete set of extension icons for Chrome Web Store and browser toolbar display.

**What this builds:** Professional icon set at all required sizes with active/inactive states.

**Why it matters:** Icons are the extension's visual identity and required for Chrome Web Store submission.

## Requirements

- TEST-07: Extension icon set created (16px, 48px, 128px)

## Tasks

### Task 1: Design Extension Icon

**Goal:** Create a distinctive icon that represents "Claude styling on other sites".

**Implementation:**
1. Design concept: Stylized "C" or Claude-inspired element
2. Use Claude color palette (orange/coral accent)
3. Ensure recognizable at 16px
4. Create clean, modern design
5. Export as SVG master

**Design considerations:**
- Must work on both light and dark browser themes
- Should suggest "markdown" or "styling"
- Avoid text (won't be readable at 16px)
- Simple geometric shapes work best

**Files to create:**
- Create: icons/icon-master.svg

**Commit:** feat(05-04): create extension icon design

### Task 2: Generate PNG Icons at All Sizes

**Goal:** Export PNG icons at required sizes for Chrome extension.

**Implementation:**
1. Export 16x16px (toolbar)
2. Export 48x48px (extensions page)
3. Export 128x128px (Chrome Web Store)
4. Ensure crisp edges at all sizes
5. Optimize file sizes

**Files to create:**
- Create: icons/icon16.png
- Create: icons/icon48.png
- Create: icons/icon128.png

**Commit:** feat(05-04): generate PNG icons at required sizes

### Task 3: Create Inactive/Disabled State Icons

**Goal:** Create grayscale versions for when extension is disabled on a site.

**Implementation:**
1. Create grayscale variants of each size
2. Ensure they're clearly "off" state
3. Update manifest.json to reference inactive icons
4. Update background.js to set inactive icon when disabled

**Files to create:**
- Create: icons/icon16-inactive.png
- Create: icons/icon48-inactive.png
- Create: icons/icon128-inactive.png

**Files to modify:**
- Modify: manifest.json (add inactive icon paths if needed)
- Modify: background.js (setIcon for inactive state)

**Commit:** feat(05-04): add inactive state icons

### Task 4: Update Manifest with Icons

**Goal:** Ensure manifest.json references all icon sizes correctly.

**Implementation:**
1. Add icons section to manifest.json
2. Verify paths are correct
3. Test icon display in Chrome

**Files to modify:**
- Modify: manifest.json

**Commit:** fix(05-04): update manifest with icon references

## Success Criteria

- [ ] Icons exist at 16px, 48px, and 128px
- [ ] Icons display correctly in Chrome toolbar
- [ ] Inactive state icons show when disabled
- [ ] Icons are recognizable at all sizes
- [ ] Manifest.json references all icons correctly

## Dependencies

- manifest.json must exist from Phase 1

## Notes

- Chrome caches icons aggressively; may need to reload extension
- Consider creating a 32px version for retina displays
- SVG master allows easy resizing for future needs
