# Plan 01-02: Create Claude Markdown CSS

## Objective
Create content/claude-markdown.css with scoped styling rules for all markdown elements using CSS variables from claude_index.css.

## Context
The CSS file must:
- Use CSS custom properties from claude_index.css (light + dark modes)
- Be scoped under `.claude-styled` class to avoid breaking site UI
- Cover all markdown elements: headings (h1-h6), bold/strong, inline code, code blocks, lists (ul/ol), blockquotes, tables, links, horizontal rules
- Match Claude's signature style: orange-amber accents, warm cream backgrounds, generous spacing

Reference: claude_index.css contains the color palette with:
- `--primary`: Orange-amber accent (bold text, borders, bullets)
- `--background`: Warm cream page bg
- `--foreground`: Warm dark text
- `--secondary`: Soft warm bg for inline code
- `--muted` / `--muted-foreground`: Subdued elements
- `--card` / `--card-foreground`: Code block backgrounds
- `--border`: Table/blockquote borders
- `--font-sans`, `--font-mono`: Typography

## Tasks

### Task 1: Create content/claude-markdown.css
**Type:** implementation
**File:** content/claude-markdown.css

Create CSS file with scoped rules under `.claude-styled`:

1. **Base typography**
   - Set font-family to var(--font-sans)
   - Set color to var(--foreground)
   - Set line-height to 1.6-1.75

2. **Headings (h1-h6)**
   - Warm dark foreground (var(--foreground))
   - Heavier font weight
   - Extra margin-top and margin-bottom
   - h1 largest, decreasing sizes for h2-h6

3. **Bold/Strong text**
   - Color: var(--primary) for orange-amber accent
   - Font-weight: 600 or 700

4. **Inline code**
   - Background: var(--secondary) (warm cream)
   - Border: 1px solid var(--border)
   - Border-radius: var(--radius) or 0.375rem
   - Font-family: var(--font-mono)
   - Padding: 0.125rem 0.25rem
   - Font-size: 0.875em

5. **Code blocks (pre)**
   - Background: var(--card) (dark in dark mode, light in light)
   - Border: 1px solid var(--border)
   - Border-radius: var(--radius)
   - Padding: 1rem
   - Overflow-x: auto
   - Font-family: var(--font-mono)
   - Font-size: 0.875rem
   - Line-height: 1.5

6. **Lists (ul/ol)**
   - Padding-left: 1.5rem
   - Margin-top/bottom: 0.5rem
   - Bullet/number color: var(--primary) (orange)
   - List item margin: 0.25rem 0

7. **Blockquotes**
   - Border-left: 3px solid var(--primary)
   - Background: var(--muted) or transparent
   - Padding-left: 1rem
   - Margin: 1rem 0
   - Font-style: italic (optional)

8. **Tables**
   - Border-collapse: collapse
   - Width: 100%
   - Border: 1px solid var(--border)
   - Header background: var(--muted)
   - Cell padding: 0.5rem 0.75rem
   - Alternating row backgrounds (optional)

9. **Links**
   - Color: var(--primary)
   - Text-decoration: underline or none with hover underline
   - Hover: opacity or color shift

10. **Horizontal rules**
    - Border: none
    - Border-top: 1px solid var(--border)
    - Or subtle gradient using var(--border)
    - Margin: 1.5rem 0

11. **Paragraphs**
    - Margin: 0.75rem 0
    - Line-height: 1.6

### Task 2: Support dark mode
**Type:** implementation
**File:** content/claude-markdown.css

Ensure all color references use CSS variables that change in `.dark` mode:
- All rules should automatically adapt because they reference vars like var(--primary)
- The .dark class on a parent element will switch the variable values
- No hardcoded colors anywhere
